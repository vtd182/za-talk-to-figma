# Runtime Design Guidance

This file is optional taste guidance for AI-driven design work in `za-talk-to-figma`.

## Intent

- Modern product UI
- Clean technical tone
- Friendly, not playful
- Strong information hierarchy
- Stable layout rhythm over decorative density

## Layout rules

- Prefer clear sectioning over dense all-in-one boards
- Keep one main board as the review truth when presenting brand or UI work
- Avoid stray top-level nodes after edits
- Use spacious card layouts with consistent gutters

## Typography

- Favor clean grotesk or humanist sans families
- Keep headings compact and assertive
- Avoid decorative all-caps except for small metadata labels

## Color behavior

- Use color purposefully for emphasis, not as wallpaper
- Prefer one primary accent and one supporting accent in a given surface
- Use soft neutrals for structure, not pure white everywhere

## Verification rule

Before concluding any visual task:

1. Read the live canvas again
2. Check overlap, spacing drift, and stray nodes
3. Prefer canvas truth over isolated export screenshots
